1、API网关都应该具备什么功能？
    性能： API高可用，负载均衡，容错机制
    安全： 鉴权（权限身份认证）、脱敏，流量清洗，后端前面（保证全链路可信调用），黑名单（非法调用的限制）
    日志： 日志记录（spain id, trace id）一旦涉及分布式，全链路跟踪必不可少
    缓存： 数据缓存
    监控： 记录请求响应数据，api耗时分析，性能监控，报警。
    限流： 流量控制，错峰流控，目前有漏桶算法、令牌桶算法也可以定制限流规则。
    熔断： 
    灰度： 线上灰度部署，可以减少风险
    路由： 路由是API网关最核心的模块功能，根据请求信息，转发到对应的微服务上。动态路由规则。
    静态： 代理
    协议转换： 如 HTTP -> RPC协议
